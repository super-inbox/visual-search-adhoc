## D. 为什么 39.9 / 58.2（报告 43）看起来高但合理

### Bing Images 为何平均接近 40 个芯片

Bing 图片搜索页面上方横向滚动的"相关搜索芯片"（related search chips）并非 Bing 专为单条 query 手动配置，而是算法批量生成的搜索建议导航。  

- Bing 的相关搜索芯片设计为多行/横向无限滚动，覆盖**同义词变体、风格细化、场景细化、相关主题**等维度；  
- 在我们采集的 58 条 query 中，Bing 几乎对所有 query 返回一致的上限（40 个芯片），只有极少数 query（39 个）未达上限；  
- 这 39.9 不是人工打标的"分类数量"，而是平台真实对用户暴露的结构化导航入口数量。  

### Canva 为何模板筛选芯片数偏高

Canva 模板搜索页左侧筛选面板包含多个维度：**模板格式（Poster / Presentation / Instagram Post / …）、风格（Minimalist / Colorful / …）、色调、尺寸、受众、场景**等。  

- 对英文 query，Canva 显示完整筛选面板，含 59 个可点击类目芯片（当前采集数据中大多数 ok query 均命中 59 的上限）；  
- 这 59 个筛选项是 Canva 预设的**平台级结构化分类体系**，而非对某条 query 定制生成的——它们始终在场，用户通过这些入口缩小搜索范围；  
- 59 的数量体现了 Canva 作为设计平台拥有丰富的跨场景类目覆盖（不同内容格式 × 不同使用场景 × 不同视觉风格）。

### 对比结论

这些数字是"**平台可见的结构化入口数量**"，不是主观打分。数量多说明外部平台提供了更丰富的分类导航；数量少（如 Curify 均值 8.8，Pinterest 均值 0.6）说明当前系统的结构化分类覆盖更窄或受技术限制（登录墙）。

### Canva 差异详细说明

**原报告写 43，当前从 `observations.csv` 计算得 58.19**。可能原因：

1. **不同数据快照**：原报告可能基于更早期的抓取运行，彼时 Canva 筛选面板展示的芯片数更少（Canva 曾调整 UI）；  
2. **计数口径差异**：原报告可能统计的是"侧边栏类目组数"（如 Format / Style / Color / Size 等 6-8 个大类）而非具体芯片数，再乘以每类平均展开数；  
3. **包含 login_required 查询**：若原报告将 21 条登录墙 query 按 0 计入均值，则全集均值 = (21×0 + 23 + 30 + 35×59) / 58 ≈ 36.5，仍不等于 43；  
4. **部分查询独立计数**：若原报告使用了不同子集（如仅 ok 英文 query 中随机抽样），可能恰好抽到了较多显示 30 个的查询，拉低均值。

## G. 总结

> 关于 Bing Images / Canva 平均标签数偏高的问题，说明如下：
>
> **Bing Images 均值 39.9**：数字来自对 58 条测试 query 在 Bing 图片搜索页面实际抓取的相关搜索芯片数量，逐条计数后取平均。Bing 设计上对大多数 query 展示 40 个横向滚动芯片（相关词/风格/场景建议），是平台的标准行为，不是我们的主观估计。
>
> **Canva 当前实测均值 58.2（原报告写 43）**：数字来自 36 条可正常访问的 query 上 Canva 模板搜索页侧边栏的筛选类目数量。Canva 在英文 query 下显示约 59 个格式/风格/尺寸筛选选项。原报告写 43 与当前实测存在差距，可能由于采集时间不同或早期口径不一，当前数据可复现的值是 58.2。
>
> 两个数字都高，反映的是这两个平台本身提供的**结构化分类导航比 Curify 更丰富**，这也正是分析报告想说明的：Curify 目前 8 个意图集群是清晰但覆盖偏窄的，有提升空间。
